## 1. Requirements (Context)
- **Goal**: 在小骑士头上显示一列3个箭头，模拟节奏小游戏
- **In-Scope**: 
  - 在玩家(Knight)头顶显示3个方向箭头
  - 监听键位设置中的移动键(Left/Right/Up/Down)
  - 检测玩家输入是否匹配最底部的箭头
  - 正确时箭头向下滚动生成新箭头
- **Out-of-Scope**: 分数系统、连击系统、音效

## 1.5 Code Map (Project Topology)
- **Entry Point**: `RhythmKnight.cs` - Mod主类，Initialize中添加游戏钩子
- **Input Access**: `InputHandler.Instance.inputActions` (HeroActions类)
  - `HeroActions.left`, `HeroActions.right`, `HeroActions.up`, `HeroActions.down`
- **Player Reference**: `HeroController.instance.gameObject`
- **UI Rendering**: 使用SpriteRenderer创建跟随玩家头部的世界空间UI
- **Assets**: `assets/right-arrow.png` (通过旋转获得4个方向)

## 2. Architecture
- **Pattern**: MonoBehaviour组件模式
- **Arrow Direction Enum**: Up, Down, Left, Right
- **Arrow Display**: 使用SpriteRenderer数组，垂直排列在玩家头顶
- **Input Check**: 每帧检查玩家输入，使用WasPressed检测单次按键

## 3. Detailed Design & Implementation

### 3.1 Data Structures & Interfaces
- `File: RhythmKnight.cs` (已实现)
  - 枚举 `ArrowDirection`: Up, Down, Left, Right
  - 类 `ArrowGame` 继承 `MonoBehaviour`:
    - `Sprite[] arrowSprites`: 4个方向的箭头Sprite
    - `SpriteRenderer[] arrowRenderers`: 3个SpriteRenderer组件
    - `ArrowDirection[] currentArrows`: 当前显示的3个箭头方向
    - `float ArrowSpacing`: 箭头间距 = 0.4f
    - `float HeightOffset`: 玩家头顶偏移 = 2.2f
